# SUPERIOR ECHO LINE 對話流程設計
> 版本：v1.0 | 更新：2026-05-29

---

## 整體對話狀態機（State Machine）

```
客戶第一筆訊息
      ↓
  [State: NEW]
      ↓
  ECHO: 問候 + 詢問稱呼（STRICT 第二句）
      ↓
  [State: WAITING_NAME]
      ↓
  客戶回覆名字 → 存入 CRM 客戶名稱
      ↓
  [State: MENU_SHOWN] ── 發送服務選單字卡 ──────────────────┐
      ↓                                                        │
  ┌───────────────────────────────────────────────────────┐   │
  │ 🔍 車輛凹痕評估  │  📍 分店資訊  │  📞 客訴處理     │   │
  └───────────────────────────────────────────────────────┘   │
        ↓                    ↓                 ↓              │
  [DENT_ASSESS]       [BRANCH_INFO]       [COMPLAINT]         │
        ↓                    ↓                 ↓              │
    Step 3-6          直接回答即可         轉真人處理          │
```

---

## 完整對話步驟（車輛評估路徑）

### Step 1｜觸發
- 客戶傳任何訊息 → ECHO 開始對話
- 判斷是否新客戶（LINE userId 不在 CRM）

### Step 2｜詢問稱呼（STRICT 必執行）
```
ECHO 回覆：
「您好！歡迎光臨卓越凹痕修復中心 SUPERIOR PDR 🙌
請問我該怎麼稱呼您呢？」
```
→ 客戶回覆 → 儲存 `客戶名稱` 至 Airtable

### Step 3｜顯示服務選單字卡
→ 發送 `01_service_menu.json`
→ 等待客戶點選

---

### 車輛評估路徑

#### Step 3A｜問廠牌型號
```
ECHO：「{名稱} 您好！請問您的愛車廠牌和型號是什麼呢？
例如：Toyota Camry、Tesla Model Y、BMW 5系列」
```
→ 儲存至 CRM 車輛資料：廠牌 + 型號

#### Step 4｜問年份
```
ECHO：「請問是哪一年的車呢？」
```
→ 儲存 年份

#### Step 5｜分店選擇（選完後自動派單）
→ 發送 `02_branch_select.json`（分店選擇字卡）
→ 客戶選擇分店後：
  1. 儲存 `指定分店` 至 CRM
  2. **自動觸發 Telegram 派單通知給店長**
  3. 更新客戶 `服務進度` → 已派單

#### Step 6｜詢問照片
```
ECHO：「感謝您選擇 {分店名稱}！
接下來請您提供愛車損傷部位的照片，
讓我們的專業技師初步評估 📸

可以多張上傳，角度越多越準確喔！」
```

#### Step 6B｜等待照片 + 補充照片
- 客戶上傳照片後，ECHO 詢問是否還有補充
```
「照片已收到！還有其他角度的照片要補充嗎？
（沒有的話請回覆「沒有了」或「完成」）」
```

#### Step 7｜AI 照片分析 → 顯示服務字卡

**分析結果 A：無掉漆**
```
「感謝您提供的照片，我們的初步評估如下 👇
您的烤漆狀況良好，以下是適合您的服務方案：」
```
→ 發送 `03_service_no_paint.json`

**分析結果 B：有掉漆**
```
「感謝您提供的照片 📸
您的烤漆看起來已有受損，我們提供以下服務方案：」
```
→ 發送 `04_service_paint.json`

**分析結果 C：保險桿損傷**
```
「感謝您提供的照片 📸
保險桿屬於塑膠材質，需依實際狀況評估處理方式，
以下是我們提供的服務選項：」
```
→ 發送 `05_service_bumper.json`

#### Step 8｜客戶選擇服務
→ 儲存 `服務類型` 至案件記錄
→ 店長收到 Telegram 更新通知
→ 案件狀態：待報價

## 客訴處理路徑

```
客戶點選「客訴處理」
     ↓
ECHO：「非常抱歉造成您的困擾 🙏
我已為您轉接到真人客服，將有專人盡快與您聯繫。
請問可以告訴我案件編號或您的姓名嗎？」
     ↓
通知 Telegram 群組 + 暫停 ECHO 回覆
```

## 後端 Server 需要的狀態機

### 對話狀態（存在 Redis 或 Airtable）

| State | 觸發條件 | ECHO 動作 |
|-------|---------|-----------|
| `NEW` | 第一筆訊息 | 問候 + 詢問稱呼 |
| `WAITING_NAME` | 等待名字 | 接收名字 → 存 CRM → 顯示選單 |
| `MENU_SHOWN` | 選單已顯示 | 等待點選 |
| `WAITING_BRAND` | 點選車輛評估 | 接收廠牌型號 → 存車輛 |
| `WAITING_YEAR` | 廠牌已輸入 | 接收年份 |
| `WAITING_BRANCH` | 年份已輸入 | 顯示分店字卡 |
| `WAITING_PHOTO` | 分店已選 | 接收照片 |
| `PHOTO_SUPPLEMENT` | 照片已收 | 等待補充照片 |
| `ANALYZING` | 收到完成指令 | 呼叫 AI 分析 |
| `SERVICE_OPTIONS` | 分析完成 | 顯示服務選項字卡 |
| `SERVICE_SELECTED` | 服務已選 | 更新案件 + 通知店長 |

### 真人介入規則
- 真人在 LINE OA 後台回覆 → ECHO 暫停 10 分鐘
- 10 分鐘無真人回覆 → ECHO 恢復
- 客戶主動再發訊息 → 持續通知店長
